【三棵树机制】

Widget Tree (配置树)
├── 特点: 不可变、轻量、可频繁重建
├── 职责: 描述UI应该是什么样子
└── 类比: 建筑图纸

Element Tree (生命周期树)
├── 特点: 可复用、管理生命周期
├── 职责: Widget和RenderObject的桥梁
└── 类比: 施工队长

RenderObject Tree (渲染树)
├── 特点: 重量、性能关键
├── 职责: 实际的布局和绘制
└── 类比: 实际施工

【Layout布局机制】

1. Constraints go down
   父 Widget → 子 Widget
   传递约束条件（最小/最大宽高）

2. Sizes go up
   子 Widget → 父 Widget
   返回确定的尺寸

3. Parent sets position
   父 Widget 决定子 Widget 的位置
   子 Widget 无法决定自己的位置

【约束类型】

Tight Constraints (严格约束)
  min == max
  子Widget必须使用精确尺寸
  示例: SizedBox(width: 100, height: 100)

Loose Constraints (宽松约束)
  min == 0
  子Widget可以自由选择尺寸
  示例: Center、Align

Unbounded Constraints (无界约束) ⚠️ 危险
  max == infinity
  必须给子Widget明确尺寸
  示例: ListView、Row、Column
